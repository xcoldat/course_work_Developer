# course_work_Developer
# Информационно-аналитическая система для компании-разработчика ПО

Курсовая работа по дисциплине «Информационные технологии в профессиональной деятельности».  
**Вариант 20** — «Компания разработчик программного обеспечения».

Программа реализует три основные функциональные задачи:
- оценку технического уровня (качества) программных продуктов (ERP, CRM, СУБД);
- прогноз динамики показателей (например, объём продаж, стоимость лицензий);
- оптимизацию распределения ресурсов (бюджета между проектами).

Также предусмотрена обобщённая визуализация (дашборд), демонстрирующая возможности системы.

## Требования

Программа разработана на языке **Python 3.8+** и использует следующие библиотеки:

- `tkinter` — графический интерфейс (входит в стандартную поставку Python);
- `pandas` — работа с табличными данными;
- `numpy` — математические вычисления;
- `matplotlib` — построение графиков;
- `scipy` — решение задач линейного программирования.

Установить необходимые библиотеки можно командой:
pip install pandas numpy matplotlib scipy

Код программы выполнен в одном файле для удобства проверки и содержит следующие классы:

## Описание модулей

### 1. Оценка технического уровня (`EvaluationWindow`)

Позволяет:
- выбрать тип продукции: **ERP**, **CRM** или **СУБД**;
- загрузить характеристики образцов из CSV-файла или ввести их вручную;
- добавить несколько образцов для сравнения;
- рассчитать относительные показатели относительно эталонных значений;
- построить:
  - **радиальную диаграмму** (лепестковую) с наложением характеристик всех образцов;
  - **столбчатую диаграмму** значений технического уровня (отсортированную по убыванию).

**Эталонные значения** (заданы в коде):
- *ERP*: модульность=10, отклик=100 мс, макс. пользователей=1000, надёжность=99.9%, стоимость=5000 тыс. руб.
- *CRM*: интеграции=20, синхронизация=2 с, объём=100 ГБ, удобство=9.5, цена=3000 тыс. руб.
- *СУБД*: tps=5000, стандарты SQL=5, масштабируемость=10, безопасность=4, цена=4000 тыс. руб.

### 2. Прогноз динамики показателей (`ForecastWindow`)

Реализует прогнозирование методом **линейной регрессии** (полином 1-й степени).  
Действия пользователя:
- загрузить CSV-файл с колонками `period` и `value`;
- указать количество будущих периодов;
- получить график с историческими данными, линией тренда и точками прогноза;
- увидеть численные значения прогноза в отдельном окне.

### 3. Оптимизация распределения ресурсов (`OptimizationWindow`)

Решает **задачу линейного программирования** с помощью `scipy.optimize.linprog`.  
Постановка задачи: распределить общий бюджет между тремя проектами (ERP, CRM, СУБД) так, чтобы максимизировать суммарную прибыль при ограничениях:
- минимальные и максимальные вложения в каждый проект;
- общая сумма затрат не превышает заданный бюджет.

Пользователь вводит:
- общий бюджет (тыс. руб);
- прибыльность каждого проекта (условные единицы прибыли на 1 тыс. руб вложений);
- минимальные и максимальные суммы вложений.

После решения выводится текстовый отчёт с оптимальными значениями и максимальной прибылью.

### 4. Визуализация данных (дашборд) (`DashboardWindow`)

Демонстрационный модуль, который показывает три графика:
- пример радиальной диаграммы оценки качества ERP-систем;
- пример прогноза продаж;
- пример оптимального распределения бюджета.

Также выводится краткая информация о программе.

## Примеры использования

### Оценка технического уровня

1. Откройте модуль «Оценка технического уровня».
2. Выберите тип продукции — **ERP**.
3. Нажмите «Загрузить из CSV» и выберите файл `erp_samples.csv` (пример ниже).
4. Нажмите «Выполнить оценку».
5. Появятся две диаграммы: радиальная и столбчатая.

Пример содержимого `erp_samples.csv`:

Название,Модульность,Время отклика,Макс пользователей,Надёжность,Стоимость <br>
ERP Сириус,9,110,950,99.5,4800<br>
ERP Вектор,8,95,1100,99.7,5200<br>

### Прогноз
Пример содержимого `sales.csv`:
period,value<br>
Янв,120<br>
Фев,135<br>
Мар,148<br>
Апр,160<br>
Май,175<br>
Июн,190<br>

Укажите количество периодов, чтобы получить прогноз на первые n месяцев из файла

### Оптимизация
Введите параметры:

1. Общий бюджет: 1000<br>
2. Прибыльность: 5, 4, 6<br>
3. Мин. вложения: 100, 100, 100<br>
4. Макс. вложения: 600, 600, 600<br>

Результат покажет оптимальное распределение и максимальную прибыль

## Формат входных данных
Все файлы должны быть в кодировке UTF-8 и иметь расширение .csv. Разделитель — запятая.
Для оценки: первая колонка — название образца, остальные — значения характеристик (столько же, сколько в выбранном типе продукции).
Для оценки: первая колонка — название образца, остальные — значения характеристик (столько же, сколько в выбранном типе продукции).
